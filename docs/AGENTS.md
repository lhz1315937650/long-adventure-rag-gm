# Agent 系统说明

本项目不是单一聊天提示词，而是一个多职责 Agent 系统。

Agent 契约定义在：

```text
agents/novel-gm-agents.json
```

## Agent 列表

### GM 主持 Agent

负责运行玩家行动后的下一轮小说 GM 回合。

它处理：

- 场景正文。
- NPC 即时反应。
- 战斗、探索、社交裁定。
- 世界状态变化。
- 可选行动。
- 长期记忆条目。

它绝不能替玩家说话、行动或选择立场。

### 记忆整理 Agent

负责把原始会话压缩成长期摘要。

它保留：

- 玩家明确选择。
- GM 已裁定后果。
- NPC 关系。
- 势力变化。
- 能力、物品和未解决冲突。

它不能新增未发生剧情。

### 资料库管理员 Agent

负责管理 `lore/` 中的已确认资料。

它提供：

- 资料保存。
- 资料检索。
- 资料冲突提醒。
- 背景缺口提醒。

它不能把未确认建议直接写入正式资料库。

### 自生长审计 Agent

负责分析提示词、规则和资料库，生成候选改进建议。

它输出：

- `prompt_patch`
- `rules_patch`
- `lore_gap`
- `consistency_warning`

它只能生成候选和补丁文本，不能直接改正式文件。

## 契约检查

运行：

```powershell
npm run check
```

会同时检查：

- 后端语法。
- 前端语法。
- Agent 契约完整性。
